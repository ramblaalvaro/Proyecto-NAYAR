Proceso

CÓMO FUNCIONA

Cuatro pasos desde la señal WiFi hasta la decisión de negocio.

01
CAPTURA

La Raspberry Pi escanea pasivamente las tramas WiFi de los dispositivos presentes en el espacio, sin necesidad de conexión ni consentimiento activo.

02
ANÁLISIS

Procesamiento en tiempo real: RSSI, conteo de dispositivos únicos, estimación de ocupación, detección de zonas activas e índice energético.

03
TRANSMISIÓN

Los datos se envían a ThingSpeak cada 30 segundos vía MQTT, almacenándose en la nube con histórico completo y API abierta.

04
DASHBOARD

Visualización en tiempo real desde cualquier dispositivo. KPIs, gráficos históricos, alertas y campos personalizados según tu sector.

Tecnología

STACK TÉCNICO

Hardware asequible, software robusto. Infraestructura desplegable en menos de una hora.

🍓
RASPBERRY PI

Sensor de bajo coste corriendo Ubuntu Server. El script Python captura tramas WiFi en modo monitor, procesa las métricas y las envía vía MQTT a ThingSpeak cada 30 segundos.

Hardware · Edge computing
☁️
THINGSPEAK

Backend IoT en la nube de MathWorks. Recibe, almacena y expone los datos via REST API con retención configurable. Compatible con MATLAB Analytics para clientes avanzados.

Cloud · IoT backend
📡
WIFI SNIFFING

Captura pasiva de probe requests en modo monitor. No requiere que los dispositivos estén conectados. Detecta presencia por la emisión natural de señales 802.11.

RF · 802.11 passive scan
🌐
DASHBOARD WEB

Frontend 100% estático alojado en GitHub Pages. Sin servidores propios, sin mantenimiento de backend. Actualización automática cada 30s vía fetch a la API de ThingSpeak.

Frontend · GitHub Pages
